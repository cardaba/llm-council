## Overview

Converted the council from "fresh-deliberation only" into a tool that ALSO accepts externally-generated deep research as input and critiques it — closing simultaneously the v1.0 backlog (persistence completeness, cost observability, mobile responsive, minimum local test suite). Single-user, local-only, BYOK posture preserved throughout; zero CI added per explicit D-04b lock.

Shipped end-to-end:
- **External Deep Research Critique** as a parallel entry point alongside the existing fresh-prompt flow.
- **Schema v1→v2 migration** so all v1.0 conversations load without `TypeError`.
- **In-conversation navigation primitives** (sticky stage headers, StageNavigationStrip, Show-more accordion, BackToTop) for legible long deliberations.
- **Persistence completeness** — `label_to_model` + `aggregate_rankings` now hydrate from disk on reload.
- **Cost analytics dual-column** — per-message MessageHeader line + sidebar footer (OpenRouter cap / Upstream BYOK).
- **Settings panel** with gear icon — theme, font-size, density, `stage4_threshold` slider.
- **Mobile responsive ≤768px** — native `<dialog>` drawer, `useTouchSwipe` hook with edge-zone + vertical-dominance gates, viewport-fit + safe-area-inset.
- **Visual regression** — 16 Playwright baselines (8 surfaces × 2 themes), 5 structural anti-flake measures, native Windows execution (D-03 revised mid-milestone).
- **Test suite** — 46 pytest backend tests + 55 vitest frontend tests + README `## Testing` section.
- **Two gap closures** (06-08 frontend stale-event guard, 06-09 backend detached SSE deliberations) to fix the BLOCKER race exposed by UAT.

## Milestone Summary

### Key Decisions

- **Bundling agresivo en 3 phases (vs 6 propuestas en SUMMARY.md)** — granularity `coarse` + user directive "ir a saco". Blast-radius ordering of SUMMARY.md respected (schema/critique → observability/polish → quality lock); aggressive bundling within each bucket.
- **Phase 5 bundle mandatory** — PERS-03 schema_version had to land WITH CRIT-01..08 (PITFALLS.md §CRIT-3) to avoid TypeError cascade on every v1.0 conversation reload.
- **`metadata` opaque-dict contract** — no signature change to `add_assistant_message` across Phase 5/6 extensions. All new payloads (label_to_model, aggregate_rankings, cost, profile, models, chairman, critic, stage4_triggered) ride inside the dict.
- **Cost display dual-column** — OpenRouter fee + upstream BYOK shown together; static post-deliberation (no animated ticker — anti-pattern AA2 violates Direction A "calmo").
- **Settings: native `<dialog>` + `showModal()`** — no `focus-trap-react`, no hand-rolled trap. `inert` on main content. Pattern reused in Phase 7 for mobile sidebar drawer (no rework needed).
- **D-03 revised mid-Phase-7 — Playwright runs natively on Windows.** Original D-03 mandated Docker-only. Single-user no-CI posture made cross-OS guarantee theoretical; Docker tagged as v2.1+ contingency only. Saved ~1 setup session.
- **No CI in v2.0 (D-04b)** — three test commands (pytest / vitest / playwright) documented in README only. CI tagged as v2.1 backlog.
- **06-08 / 06-09 gap closures** — UAT BLOCKER (race condition switching conversations mid-stream) closed by both surfaces: frontend guard (defensive `if (!lastMsg?.loading) return prev;` at 10 SSE-event handlers) AND backend detached tasks (deliberation survives client disconnect). Pre-existing race since SSE streaming landed (v1.0), surfaced by Phase 6 UAT.
