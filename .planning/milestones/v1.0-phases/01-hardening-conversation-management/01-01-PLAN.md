<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/SPA → FastAPI (loopback) | Path-param `conversation_id` es input no confiable que cruza al storage layer. |
| FastAPI handler → storage.get_conversation_path | Frontera donde se debe rechazar input malformado; sin validación, `os.path.join` permitía path traversal (Vuln 2). |
| storage layer → filesystem (`data/conversations/`) | Sin validación, `..\..\etc\passwd` o `..\..\Windows\system.ini` quedan al alcance del proceso del backend. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering / Information Disclosure | `backend/storage.get_conversation_path` | mitigate | `uuid.UUID(conversation_id)` rechaza cualquier input que no sea UUID parseable; ningún UUID válido puede contener `/`, `\`, `..`. Cierra Vuln 2 documentada en CONCERNS.md. |
| T-01-02 | Information Disclosure | `backend/main.py` handlers — leakage de detalles internos | mitigate | El detail del 400 es la cadena fija "Invalid conversation ID" — no expone el ValueError original ni el path intentado. |
| T-01-03 | Denial of Service | Constructor `uuid.UUID()` con inputs anómalos | accept | El parser stdlib es O(1) y no hace I/O. No se ha documentado CVE de DoS contra `uuid.UUID()`. Riesgo inexistente. |
| T-01-04 | Elevation of Privilege | Backend bind a `0.0.0.0` accidental | accept | Bind explícito a `127.0.0.1:8001` en `main.py:199`. No es alcanzable desde otra máquina. Fuera del alcance de SEC-01 (corresponde a Vuln 1, ya cerrada). |
| T-01-05 | Repudiation | Sin auditoría de delete | accept | Single-user local; deferred a v2 si surge necesidad. Sin requirement asociado en este milestone. |
| T-01-06 | Spoofing | Falsificación de identidad de cliente | accept | Single-user local sin auth (PROJECT.md Constraints). Modelo de amenaza: "otro proceso en la misma máquina"; mitigado por bind a loopback. |

**Disposiciones aceptadas (T-01-03..06)** son consistentes con Constraints en PROJECT.md (single-user, local-only, no auth).
</threat_model>
