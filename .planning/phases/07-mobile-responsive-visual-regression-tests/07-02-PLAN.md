<objective>
Ship the mobile drawer + swipe gesture per MOBL-02 and MOBL-04. The drawer mirrors the Phase 6 `SettingsPanel.jsx` `<dialog>` + `showModal()` + ESC-dismiss pattern verbatim (left-anchored instead of right, full viewport height, soft Direction A backdrop, mobile-only via media query). The swipe hook (~32 LOC) opens the drawer on left-edge swipe-right and closes it on swipe-left, with non-negotiable vertical-dominance rejection and edge-zone gate so scrolling messages never opens the drawer. A hamburger button is added to the Header (visible only at ≤768px) for the tap-to-open path.

Purpose: D-01 locks MOBL-04 to ship — no scope-cut. Both behaviours land in the same plan because the hamburger button, drawer, and swipe hook share state (`drawerOpen` in App.jsx) and are meaningful only together.

Output: A fully functional left-edge mobile drawer with both tap and swipe activation, behind a media query so desktop is untouched.
</objective>

Create `frontend/src/hooks/useTouchSwipe.js` (~32 LOC) per RESEARCH Recipe 5 and PATTERNS.md hook section. The exact body:

- Imports: `import { useRef, useCallback } from 'react';`
- Module-level constants (UPPER_SNAKE per CLAUDE.md backend convention applied here for parity with `useSettings.js` KEY_*):
  - `const DEFAULT_THRESHOLD = 40;` (// px — iOS Safari back-swipe default)
  - `const EDGE_ZONE = 24;` (// px — only swipes starting in this left strip count as drawer-open candidates)
- Module docstring (above the function, JSDoc style per PATTERNS.md "Module header docstring pattern" lines 186-200) summarising: contract, `touch-action: pan-y` requirement on `.messages-container`, passive-listener constraint (no `preventDefault()`), stable handler discipline, public API.
- Export: `export function useTouchSwipe({ isOpen, onOpen, onClose, threshold = DEFAULT_THRESHOLD }) { ... }`.
- Three `useRef` cells: `startX`, `startY`, `startedAtEdge` (initial `null` / `false`).
- Three `useCallback` handlers:
  - `onTouchStart` (deps: `[]`) — read `e.touches[0]`, store clientX/clientY into refs, set `startedAtEdge.current = t.clientX <= EDGE_ZONE`.
  - `onTouchMove` (deps: `[]`) — explicit no-op with a comment explaining "passive listener can't preventDefault; reserved for future cap" (RESEARCH Recipe 5).
  - `onTouchEnd` (deps: `[isOpen, onOpen, onClose, threshold]`) — read `e.changedTouches[0]` (NOT `e.touches[0]`), compute dx/dy, reject if `Math.abs(dy) > Math.abs(dx)` (vertical-dominance branch — RESEARCH explicitly calls this out as the single most important line), then check open/close conditions, then reset `startX.current = null`.
- Return `{ onTouchStart, onTouchMove, onTouchEnd }`.

Preserve the 4 landmines as inline comments inside the hook (PATTERNS.md lines 214-218 enumerates them; RESEARCH Recipe 5 prints them as a list — collapse into terse one-line `// ` comments above the relevant code lines):
1. Passive listeners by default — do not call `preventDefault()`.
2. Scroll-direction conflict — the `Math.abs(dy) > Math.abs(dx)` rejection is non-negotiable.
3. `changedTouches` vs `touches` — `onTouchEnd` must read `e.changedTouches[0]`.
4. Edge zone — only count gestures that *start* in the left 24px as drawer-open candidates.

No localStorage. No re-renders. No state cells (refs only).
