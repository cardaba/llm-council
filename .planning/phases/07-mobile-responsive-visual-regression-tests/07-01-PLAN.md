<objective>
Lay the responsive substrate that the rest of Phase 7 builds on. This plan introduces the `--touch-target-min: 44px` CSS variable, switches the viewport meta to `viewport-fit=cover`, and threads `env(safe-area-inset-*)` into the Header, composer, and main app shell using the `max(env(...), fallback)` idiom so non-notched devices keep their existing padding rhythm.

Purpose: VRT (07-03) cannot baseline a moving target — every interactive control must be at its final mobile geometry before snapshots are taken. The drawer (07-02) also depends on the `--touch-target-min` token and the `viewport-fit=cover` meta being in place.

Output: Mobile-touch-target-compliant interactive controls at ≤768px and safe-area-aware Header / composer padding, with zero visual change on desktop.
</objective>
