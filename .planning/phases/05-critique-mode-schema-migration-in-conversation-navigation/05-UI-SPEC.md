Inherited verbatim from `frontend/src/index.css:86-93` ‚Äî 4px-base, 7 tokens. All Phase 5 styles MUST consume these tokens; no hardcoded px values except where noted under "Exceptions".

**Exceptions** (declared, justified, isolated):

## Interaction Contracts

### Critique welcome state ‚Äî submit eligibility

| Condition | Submit state |
|-----------|--------------|
| 0 files AND textarea empty | Disabled (`opacity: 0.5; cursor: not-allowed`) ‚Äî tooltip: locked copy above |
| 0 files AND textarea non-empty (whitespace-trimmed) | Disabled ‚Äî same tooltip |
| ‚â•1 file AND textarea empty | Disabled ‚Äî same tooltip |
| ‚â•1 file AND textarea non-empty | Enabled (primary accent CTA) |
| Total estimated tokens >150K | Disabled + inline `--color-error` message (CRIT-06 copy locked above) |
| 5 critiques launched in the last hour (localStorage) | Enabled, but click opens re-confirmation Modal (CRIT-07 copy locked above) |

### File slot interactions (D-04, D-05, D-06)

- **Single click** on idle drop-zone ‚Üí opens native file picker (`.md,.txt`). No drag-and-drop is REQUIRED, but the zone MUST accept drag-and-drop (`ondragover` / `ondrop`) as a progressive enhancement; dragover state shows accent outline.
- **Drag a file from OS** over a drop-zone ‚Üí outline switches to `--color-accent`, no other state changes.
- **Drop / pick a valid file** ‚Üí drop-zone enters loaded state showing the file chip (`‚úÖ filename  size_kb KB  ‚úï`). The `publisher/model-id` slot label remains visible above the chip.
- **Click ‚úï on chip** ‚Üí slot returns to idle empty state. NO confirmation dialog (D-06 ‚Äî cost of correction is 2 actions, acceptable).
- **No drag-between-slots; no caret menu on chip** (D-06 lock).
- **Wrong slot upload (per D-06)** ‚Üí user clicks ‚úï on the wrong slot, then clicks the correct slot's drop-zone and re-uploads. The file content has not yet been persisted (only held in component state), so no server round-trip.

### Stage navigation strip (NAV-02)

- **Click chip** ‚Üí smooth-scroll `.messages-container` to the target stage section. Reduced-motion: instant scroll.
- **Scroll-spy** (IntersectionObserver, root=`.messages-container`, `rootMargin: "-52px 0px -50% 0px"`, `threshold: 0`) marks the chip whose section currently occupies the top half of the viewport.
- **Active chip styling:** `border-bottom: 2px solid var(--color-accent); color: var(--color-accent); background: var(--color-accent-soft)`. Inactive chips: subtle border, `--color-fg-secondary` text, `--color-bg-secondary` background.
- **Live SSE update** ‚Äî chip text updates per-stage:
  - On `stage1_start`: chip shows `Stage 1 ¬∑ streaming‚Ä¶`.
  - On `stage1_complete`: chip shows `Stage 1 ¬∑ {N} responses` where `N = data.length`.
  - On `stage2_start`: render Stage 2 chip with `Stage 2 ¬∑ evaluating‚Ä¶` (UNLESS n=1, in which case Stage 2 chip is NEVER rendered per D-05).
  - On `stage2_complete`: chip shows `Stage 2 ¬∑ peer review`.
  - On `stage3_start`: render Stage 3 chip with `Stage 3 ¬∑ synthesising‚Ä¶`.
  - On `stage3_complete`: chip shows `Stage 3 ¬∑ synthesis`.
  - On Stage 4 (if `stage4` payload arrives in the message): render `Stage 4 ¬∑ refinement`.

### "Critique" sidebar pill (D-03)

- Renders only when `conv.mode === "critique"`.
- Position: inline next to title, after the title text, separated by `var(--space-2)`.
- The pill MUST NOT push the conversation row taller than fresh-mode rows. Achieved by `align-items: baseline` on the title row and the pill's vertical padding being `var(--space-1)` (4px).
- On rows narrow enough that the title + pill don't fit on one line: title `text-overflow: ellipsis`, pill stays right-aligned and intact. Planner may shorten to `C` only if visual testing on ‚â§320px confirms row break.

### Sticky stage headers (NAV-01)

- `position: sticky; top: 0;` on each `<h2>` (or `<h3>`) acting as a stage section header inside the deliberation.
- Scroll container is `.messages-container` (already below global Header at 52px in the grid layout). `top: 0` is correct ‚Äî do NOT add `top: var(--layout-header-h)` (RESEARCH ¬ß5).
- Header background must be opaque (`var(--color-bg-primary)`) so scrolling content does not bleed through.
- Bottom rule (`border-bottom: 1px solid var(--color-border-subtle)`) provides visual separation when pinned.

### Back-to-top button (NAV-04)

- Listens to `.messages-container` scroll position via a scroll event handler with passive listener.
- Visible when `scrollTop > 800`. Hidden otherwise via `opacity: 0; pointer-events: none`.
- Click ‚Üí `scrollTo({ top: 0, behavior: prefersReducedMotion ? 'auto' : 'smooth' })`.
- Uses `window.matchMedia('(prefers-reduced-motion: reduce)').matches` to determine behavior at click-time (so a user toggling the OS setting mid-session gets the new behavior on next click).

### Stage 1 collapse (NAV-03)

- Each Stage 1 response card measures its content height on first render (or via `ResizeObserver` if planner prefers).
- If `contentHeight > 600px` AND the conversation is **historical** (i.e. `stage3` payload is already present on first render ‚Äî not currently streaming), the card defaults to collapsed.
- Collapsed state: max ~400px visible, `Show more ‚åÑ` button at the bottom.
- Click button ‚Üí expand via `grid-template-rows: 0fr ‚Üí 1fr` accordion (same CSS pattern as `ReasoningDisclosure`).
- During **live** streaming, all Stage 1 cards stay expanded until Stage 2 starts; then non-active cards auto-collapse if >600px. The active card (the one currently streaming) stays expanded. (CONTEXT.md Claude's Discretion ‚Äî locked here.)

### File chip hydration on reload (CRIT-08)

- When a critique conversation loads (`get_conversation` returns `mode: "critique"` + `external_research` dict on the assistant message):
- `ExternalResearchPanel` renders 3 chips (or 1/2 if council collapsed) above the assistant message.
- Each chip is collapsed by default: shows `{publisher/model-id} ¬∑ {filename} ¬∑ {size_kb} KB  ‚åÑ`.
- Click chip ‚Üí expand via `grid-template-rows: 0fr ‚Üí 1fr`. Expanded body shows file content via `<Markdown>` inside `<div className="markdown-content">`.
- Chevron flips from `‚åÑ` to `‚åÉ` on expand.
- Multiple chips can be expanded simultaneously (no accordion-radio behavior).
