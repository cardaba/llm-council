# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6 (Persistence completeness + Cost analytics + Settings panel). Generated by gsd-ui-researcher from 100% pre-populated upstream artifacts (CONTEXT.md D-01..D-06 + Claude's Discretion subsection + v1.0 Phase 4 locked token system in `frontend/src/index.css`). Verified by gsd-ui-checker.

**Direction A "calmo" is the parent tone** (locked v1.0 Phase 4). No animated tickers (AA2), no onboarding tour (AA1), no LLM-powered settings explainers (AA13), no scrim/overlay darkening (D-04), no Save/Cancel buttons (D-03), no tick marks, no `+/-` buttons, no dynamic slider microcopy (D-05).

This phase surfaces three new UI features:
1. **MessageHeader cost line** — static, post-deliberation, single line, hide when `metadata.cost.total < 0.001`.
2. **Sidebar footer cost summary** — two columns (OpenRouter cap left, Upstream BYOK right), progress bar visible ≥80% of cap.
3. **Settings panel** — slide-out drawer 380px right (full-width on mobile), native `<dialog>` + `showModal()`, immediate-apply, no scrim, 4 controls.

## Motion & Interaction

**Existing v1.0 motion tokens (locked) — Phase 6 uses 3 of 4 and proposes ZERO new tokens.**

| Token | Value | Usage in Phase 6 |
|-------|-------|------------------|
| `--motion-duration-fast` | 120ms | Hover state on gear icon button (background-color tint); hover on density/font-size radio segments; click-down state on slider thumb |
| `--motion-duration-base` | 180ms | Settings panel slide-out entry transition (translate3d from `100%` to `0` on the X axis); progress bar fill width transition when usage value updates |

**Specific motion contracts:**
- **Panel entry**: `transform: translate3d(100%, 0, 0) → translate3d(0, 0, 0)` over `var(--motion-duration-base) var(--motion-easing-out)` = 180ms ease-out (per CONTEXT.md Claude's Discretion recommendation of ~200ms ease-out — 180ms is the closest in-token value; consistent with `Sidebar.css:39`).
- **Panel exit**: `transform: translate3d(0, 0, 0) → translate3d(100%, 0, 0)` over `var(--motion-duration-fast) var(--motion-easing-out)` = 120ms ease-out (slightly faster than entry per CONTEXT.md recommendation of 150ms — 120ms is closest in-token value; exit-faster-than-entry is the standard Direction A interaction pattern).
- **Progress bar fill width**: `width` transition over `var(--motion-duration-base) var(--motion-easing-out)` when the underlying usage ratio changes (only fires on data refetch, not on every render).
- **Slider thumb**: no custom transition — rely on native `<input type="range">` browser behavior + `accent-color` color binding.
- **Cost line render**: NO entry animation. Renders statically on conversation hydrate. (Per AA2 anti-feature: no animated ticker, no count-up.)
- **Sidebar footer cost summary**: NO entry animation. Renders statically on first mount + after each `GET /api/stats/cost` response.

**`prefers-reduced-motion` honored globally** via the existing `@media (prefers-reduced-motion: reduce)` block in `index.css:178-189` which clamps all transitions to 0.01ms. No additional reduced-motion logic required in Phase 6 components.

## Behavior Specifications

**Cost line render rules (COST-02 + D-01):**
- Visible when `metadata.cost.total >= 0.001`.
- Hidden (zero render — no placeholder, no `—`) when `metadata.cost.total < 0.001` OR `metadata.cost` is missing/malformed (backwards compat with pre-Phase-6 messages).
- 3-decimal precision on both `$X.XXX upstream` and `$Y.YYY fee` regardless of magnitude.
- Hierarchy: upstream is the **prominent** segment (semibold body-size + `--color-fg-primary`); fee is the **muted** segment (regular microcopy + `--color-fg-muted`). Even though upstream dominates visually, the segment order is upstream-then-fee per D-01.

**Sidebar footer render rules (COST-04 + D-02):**
- Always rendered (footer block always present once data fetched).
- Initial-mount placeholder: `Loading cost stats…` until first `GET /api/stats/cost` response.
- Two-column block visible when `current_month.queries >= 1`.
- Single-line empty-state (`No queries this month yet.`) when `queries === 0` — two-column block hidden.
- Progress bar visible **only** when `(current_month.total_usd / 100) >= 0.80`. Below threshold: the bar slot is collapsed (no empty track shown).
- Refetch on app mount, on each new assistant message complete (subscribe to the same SSE `complete` event that App.jsx already listens for), and on `visibilitychange` when document becomes visible after >5min hidden (plan-time may scope-cut visibilitychange — minimum is mount+complete refetch).

**Settings panel open/dismiss rules (SET-01 + D-04):**
- Open: gear icon click → `setSettingsOpen(true)` → `<dialog>` element's `showModal()` invoked via `useEffect` watching the boolean.
- Dismiss methods (all four MUST work):
  1. **ESC key** — native `<dialog>` behavior, no JS needed.
  2. **Click outside the panel** — listen for `click` event on the dialog's backdrop pseudo-element OR fall back to listening on the dialog itself with `e.target === e.currentTarget` guard (matches `Modal.jsx:84-89` pattern).
  3. **X close button** (panel header, top-right, 20×20 stroke SVG) — explicit dismiss.
  4. **Gear icon re-click** (Header) — toggles closed.
- **No scrim**: do NOT render `.modal-backdrop` overlay. The native `<dialog>` ::backdrop pseudo-element MUST be styled `background: transparent` to suppress the default semi-transparent overlay.
- Focus management: on open, focus moves to first focusable element inside panel (the Theme toggle — same control as Header, so the user lands on a familiar surface). On close, focus restores to the gear icon button (matches `Modal.jsx:77` pattern).
- `inert` attribute is NOT added to main content (this is the meaningful divergence from MOBL-02's mobile drawer pattern — desktop settings panel keeps main content reading-able per D-04).

**Settings controls — immediate-apply (D-03):**
- **Theme toggle**: identical to Header — uses `useTheme().toggle`. Click flips theme instantly; persists to localStorage via existing `useTheme` setter.
- **Font size radio**: click S/M/L → `useSettings().setFontSize(value)` → updates React state → applies `data-fontsize` attribute on `<html>` → root font-size cascades (single-render flicker acceptable per SET-04).
- **Density radio**: click Compact/Comfortable → `useSettings().setDensity(value)` → applies `data-density` attribute on `<html>` → spacing tokens re-resolve. **No flicker**: the FOUC blocker pre-applies on subsequent loads; the live change happens within the React commit and is visually continuous.
- **Stage 4 threshold slider**: drag → `useSettings().setStage4Threshold(parseInt(value))` → updates React state + localStorage on every `change` event (not `input` — `change` fires once per drag-release, avoiding 60Hz localStorage writes); the new value travels in the next `quality_research` request body. In-flight queries are not affected (D-03).

**Stage 4 threshold defaults & validation:**
- Initial slider position on first mount = `PROFILES["quality_research"]["stage4_threshold"]` = `8`. NO hardcoding `8` in frontend — the value must be sourced from a backend-exposed constant OR a backend-fetched config endpoint (plan-time decides whether to expose via `GET /api/config` or hardcode with a code comment pointing to the PROFILES source).
- localStorage key: `stage4Threshold` (camelCase, matches `useTheme`'s `theme` key style).
- Validation: when localStorage value is missing, non-integer, or outside `[1, 10]`, fall back to the PROFILES default.

## Visual Hierarchy & Decision Density

**Hierarchy in MessageHeader cost line (D-01):**
- Upstream = ~95% of real spend → prominent (semibold + body-size + primary fg).
- Fee = ~5% of real spend → quiet (regular + microcopy + muted fg).
- Visual weight mirrors monetary weight. The user reads the dominant cost first.

**Hierarchy in sidebar footer (D-02):**
- Cap structure is the headline — two columns make the OpenRouter / Upstream dichotomy unmissable.
- Within each column: value is prominent, label is quiet, subtext is muted.
- Progress bar is BINARY (visible/hidden) — no gradient between 0-79%. Crossing 80% is a single discrete signal, not a gradient anxiety surface.

**Hierarchy in settings panel:**
- Most-frequently-toggled at top (theme), most-impactful-on-cost at bottom (stage4_threshold) per CONTEXT.md Claude's Discretion recommendation.
- Control order: **Theme · Font size · Density · Stage 4 threshold**.
- Each control group is visually equal (same label-size heading, same vertical padding) — no control is "highlighted" or "primary". This is correct Direction A "calmo" treatment.

**Hierarchy in settings panel (the divergence from D-01):**
- D-01 says "upstream prominent, fee secondary" because the line shows real cost.
- D-02 says "two equal columns" because the line shows cap pressure.
- This is INTENTIONAL per CONTEXT.md D-02 closing sentence — the contexts differ. Plan-time should not "harmonize" these into a single layout pattern.
