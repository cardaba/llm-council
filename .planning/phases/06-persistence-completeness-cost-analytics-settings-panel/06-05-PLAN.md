<objective>
Land the FOUC-blocker extension + CSS variable overrides that make density and font-size apply synchronously before React mounts. This is the "wire the rendering substrate" task — it does not introduce a Settings panel (Plan 06 owns that), it does not introduce a `useSettings` hook (Plan 06 owns that), but it makes the data-* attributes that those upcoming surfaces will write into actually affect the rendered DOM.

Purpose: SET-04 explicitly requires density to apply via the FOUC blocker (sync, no flicker), and the v1.0/Phase-4 token system is `rem`-based so font-size root override works for free. Landing this in Wave 1 unblocks Plan 06 because the panel's setters can directly mutate `document.documentElement.setAttribute('data-density', v)` and the result is visible without any further plumbing.

Output: A user can manually run `document.documentElement.setAttribute('data-density', 'compact')` in DevTools and observe the spacing tighten. Same for `'data-fontsize'` with `s|m|l`. Plan 06's `useSettings` hook later wires these to localStorage + state.
</objective>
