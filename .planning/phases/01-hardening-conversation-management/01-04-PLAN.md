    7. **Filtro final (title o content)**:
    ```jsx
    const filteredConversations = useMemo(() => {
      if (!contentSearchActive || !contentCache) return titleMatches;
      const q = debouncedQuery.toLowerCase();
      if (!q) return conversations;
      return conversations.filter((c) => {
        // Match title first (cheap)
        if ((c.title || '').toLowerCase().includes(q)) return true;
        const full = contentCache.get(c.id);
        if (!full) return false;
        return full.messages.some((m) => {
          if (m.role === 'user') {
            return (m.content || '').toLowerCase().includes(q);
          }
          // Assistant: search stage3.response and stage1[*].response
          const s3 = (m.stage3?.response || '').toLowerCase();
          if (s3.includes(q)) return true;
          return (m.stage1 || []).some((r) =>
            (r.response || '').toLowerCase().includes(q)
          );
        });
      });
    }, [contentSearchActive, contentCache, conversations, titleMatches, debouncedQuery]);
    ```

    8. **Activar content search** (lazy load Promise.all):
    ```jsx
    const activateContentSearch = async () => {
      if (contentSearchActive || isLoadingContent) return;
      setIsLoadingContent(true);
      try {
        const fulls = await Promise.all(
          conversations.map((c) => api.getConversation(c.id))
        );
        const cache = new Map(fulls.map((c) => [c.id, c]));
        setContentCache(cache);
        setContentSearchActive(true);
      } catch (e) {
        console.error('Content search load failed:', e);
      } finally {
        setIsLoadingContent(false);
      }
    };
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/conversations/{id} (existente) | El content fallback dispara N llamadas GET en paralelo; cada una pasa por la validación UUID de Plan 01 (no nueva superficie). |
| Memoria del navegador → contentCache (Map) | ~50-500KB por sesión (D-11); aceptable. Sin persistencia a localStorage (RESEARCH §Anti-Patterns línea 588 — query es efímera). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | searchQuery enviada a localStorage / URL | mitigate | Estado solo en React (`useState`), nunca persistido. RESEARCH §Anti-Patterns línea 588 lo prohíbe explícitamente. |
| T-04-02 | DoS | Activate content search con cientos de conversaciones | accept | D-11 estimó ~10-100 conversaciones para el caso de uso personal; ~500KB. Si en el futuro la lista crece a 1000+, RESEARCH recomienda virtualization en Phase futura. Modelo de uso single-user lo hace inalcanzable hoy. |
| T-04-03 | XSS via search query renderizada | El placeholder `No matches for "${debouncedQuery}"` muestra la query | mitigate | React escapa el child string automáticamente; no se interpolan tags HTML. |
| T-04-04 | Information Disclosure via cache stale | Cache no se invalida tras delete/rename | accept | Per D-11 acepta cierta staleness. Si una conversación borrada queda en el cache, el filter aún la matchearía via contenido pero la conversación no aparece en `conversations` (lista metadata recién recargada), por lo que no se renderiza. Coherencia mantenida. |
</threat_model>
