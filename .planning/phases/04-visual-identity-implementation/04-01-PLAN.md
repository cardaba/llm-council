<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser localStorage | Atacante con acceso al mismo navegador puede manipular `localStorage.theme`. Bajo riesgo: el único efecto es flip del tema visual; no hay PII ni capability escalation. |
| Static font assets | woff2 binarios servidos por Vite desde `frontend/public/fonts/`. No ejecutables; no hay parsing de fuentes hostiles porque las descargamos de canonical SIL OFL sources. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | localStorage `theme` key | accept | Solo cambia `data-theme` attribute. No hay code execution path desde el valor. Try/catch en el inline script previene crash si `getItem` lanza (private browsing). |
| T-04-02 | Information Disclosure | Font preloads | accept | woff2 son public assets sirviendo de `/fonts/`. No hay datos sensibles en font files (SIL OFL fonts canonical from Adobe / rsms / JetBrains). |
| T-04-03 | Tampering | woff2 supply chain | mitigate | Descargar SOLO desde canonical SIL OFL sources (github.com/adobe-fonts, github.com/rsms/inter, github.com/JetBrains/JetBrainsMono); commit binarios al repo (no hot-link a CDN externo); SUMMARY.md registra source URL + size en bytes. Subsequent builds usan exactamente los binarios commiteados. |
| T-04-04 | Denial of Service | FOUC blocking script | accept | El script bloquea el primer paint por <1ms en machines modernas. localStorage acceso síncrono. matchMedia sync. Total budget <2ms. Aceptable trade vs FOUC visible (200ms de flash). |
</threat_model>
