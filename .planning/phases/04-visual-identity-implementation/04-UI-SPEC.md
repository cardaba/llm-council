### Accent reserved for (explicit allowlist — never "all interactive elements")

The terracota `--color-accent` MUST appear ONLY on:

1. **Stage 3 highlight** — `border-left: 3px solid var(--color-accent)` + background `var(--color-accent-soft)` on the `.stage3` panel. Replaces the prohibited `#f0fff0` Bootstrap green (closes VIS-01, F-12, H8-01).
2. **Primary call-to-action** — the Send button in `ChatInterface` (filled background).
3. **Active sidebar row** — `border-left: 3px solid var(--color-accent)` indicator + slight `var(--color-accent-soft)` tint.
4. **Active QualityToggle segment** — 1px `var(--color-accent)` border around the selected radio chip.
5. **Stage 1 active tab** — underline `2px solid var(--color-accent)`.
6. **Active progress-strip segment** — dots animated with `background: var(--color-accent)`.
7. **Focus rings** — `:focus-visible` outline uses `var(--color-focus-ring)` (alpha-modulated accent) on every focusable element.
8. **Inline links inside `.markdown-content`** — `color: var(--color-accent)` + underline-on-hover.
9. **Search input focus border** — `border: 1px solid var(--color-accent)` while focused.
10. **Three-dot menu hover/active item** — background `var(--color-accent-soft)` on hover.

Anything else (sidebar item hover, button hover background, Modal Cancel button, Stage 1/2 panel borders) uses NEUTRALS only — `--color-bg-secondary`, `--color-border-subtle`, `--color-fg-secondary`. The accent must stay scarce; that scarcity is what makes the "Research notebook" tone work.

- Persistent ErrorBanner background tint + border-top + retry-button border.
- Failed-model `⚠` glyph + name strikethrough in progress strip and Stage 1 tab list.
- Inline rename validation error (out of scope for v1; slot reserved).

Never on regular interactive elements.

| Token | Value | Source |
|-------|-------|--------|

### Locked microinteraction specs

| Surface | Spec |
|---------|------|
| Sidebar / button hover | `transition: background-color var(--motion-duration-base) var(--motion-easing-out)`. NO transform, NO scale. Background fades to `--color-bg-secondary` (or `rgba(176, 90, 42, 0.06)` tint on accent-adjacent). |
| Progress strip dots | 3 dots, `@keyframes dot-pulse` 660ms `ease-in-out infinite`, staggered with `animation-delay` 0 / 220ms / 440ms. Active segment only. (D-13) |
| Progress segment `done` fade | `transition: background-color var(--motion-duration-slow) var(--motion-easing-out)` to `--color-accent-soft`. |
| Progress strip auto-collapse after Stage 3 complete | `grid-template-rows: 1fr → 0fr` over `--motion-duration-slow` (or `max-height: 0`). Same trick as accordion. |
| Reasoning disclosure | `grid-template-rows: 0fr → 1fr` over `--motion-duration-slow var(--motion-easing-out)`. Inner wrapper has `overflow: hidden; min-height: 0;`. Chevron `›` rotates `transform: rotate(90deg)` over `--motion-duration-base`. (D-12) |
| QualityToggle hover | Cost subtitle fade-in `opacity 0→1` over `--motion-duration-fast`. Hovered chip gains `box-shadow: inset 0 0 0 1px var(--color-accent)` over 160ms. |
| Stage 3 reveal | `@keyframes` 1× on `opacity 0→1` + `translateY(8px → 0)` over `--motion-duration-slow`. |
| Stage 4 spinner → Stage 4 panel | Spinner sits BETWEEN Stage 3 spinner and Stage 3 panel (Phase 3 / Plan 03-05 D-23). When Stage 4 completes, spinner unmounts and the Stage-4 sub-section materialises INSIDE the Stage-3 panel — net layout shift zero. |
| Theme switch | NO transition on `data-theme` change. The cross-component flip is intentional and instant — animating it across hundreds of properties is jank-prone and CONTEXT did not request it. |
| Mobile drawer (W23) | `transform: translateX(-100% → 0)` over 200ms `ease-out` on `[data-open="true"]`. Backdrop fade unnecessary at this scope. |
| Focus rings | `outline: 2px solid var(--color-focus-ring); outline-offset: 2px;` — STATIC, not transitioned. Always visible on `:focus-visible`. |

### `prefers-reduced-motion` policy

Global override in `index.css` (RESEARCH Example 8). Reduces ALL `animation-duration` / `transition-duration` to `0.01ms !important`, but PRESERVES `:focus-visible` outline (essential, not decorative). `scroll-behavior: auto !important` to defeat smooth-scroll. (D-11, RESEARCH Pitfall 4)
