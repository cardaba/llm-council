**Captured:** 2026-05-11 during v2.0 milestone closure manual verification.
**Source:** User UX analysis on running app (post-Phase-7 build, post-MOBL/VRT/TEST).
**Status:** Raw input for next `/gsd-new-milestone` discussion. Not yet scoped, not yet prioritized into requirements.

## P0 — Regression-class (likely Phase 5 NAV-01 fix)

### Sticky stage header — ghost strip + ambiguous anchoring

**What the user saw:**
> El bloque sticky que contiene los tabs (Stage 1 · 4 responses / Stage 2 · peer review / Stage 3 · synthesis) y el título "Stage 1: Individual Responses" queda fijado en la parte superior del panel central. Sin embargo, por encima de ese sticky queda una franja visible del contenido que sigue haciendo scroll (se ve cómo asoma `println!("Start CPU {}", id);` o `import asyncio` justo arriba del sticky).

**Diagnosis (user-supplied, technically correct):**
- The sticky container and the scroll container do not share the same top edge.
- The scroll container has `padding-top` (or similar) creating a gap above the sticky's `top: 0`.
- Missing scroll-progressive feedback (shadow/border on stick).

**Fix shape (user-supplied):**
- Put `position: sticky; top: 0` on a container that wraps `tabs + title + bottom separator` as one block.
- Opaque background + `box-shadow` triggered by sentinel/IntersectionObserver on scroll.
- If the global header is also fixed, set sticky `top: var(--global-header-height)` exactly.
- Move any padding meant to separate from the global header to the LAYOUT PARENT, not inside the scroll container.

**Why this counts as a regression vs polish:** NAV-01 requirement reads "sticky stage headers". The current implementation fulfills the letter but the visual experience violates the spirit (ambiguous layering). A milestone audit would have flagged this.

## P1 — Information architecture gaps

### Redundancy: tabs + H2 say the same thing

**User observation:**
> Los tabs (Stage 1/2/3) y el título "Stage 1: Individual Responses" expresan lo mismo dos veces.

**Direction:** drop the H2, let the active tab carry the semantic.

### Lost prompt context on scroll

**User observation:**
> Al scrollear ya no se ve qué pregunta originó la respuesta ni qué modelo está hablando.

**Direction:** include conversation title (or active model name) in the sticky bar as a breadcrumb / sub-context line.

### Sidebar items lack hierarchy

**User observation:**
> Todas las conversaciones tienen el mismo peso tipográfico; "2 messages" se repite como metadato poco útil. El item activo se distingue solo por un fondo melocotón sin un indicador de borde izquierdo más fuerte.

**Direction:** active-item left border (3px accent), date grouping ("Hoy" / "Esta semana"), demote `N messages` to icon + smaller font.

## P2 — Polish

### Cost footer disconnected

**User observation:**
> `$0.00/$100 · Upstream $2.97 BYOK` está visualmente desconectado y usa una tipografía que compite con la navegación. Su significado no es autoevidente.

**Direction:** consider moving cost block out of the sidebar (popover from a "spending" icon? settings panel sub-section?). Phase 6 D-02 locked the dual-column layout — revisit that lock with v2.1 eyes.

### Scroll-to-top button: contrast + a11y

**User observation:**
> El botón "↑ scroll to top" flota en una esquina sin etiqueta accesible visible y sin contraste suficiente sobre el fondo claro.

**Direction:** `aria-label="Volver al inicio"`, raise contrast (Direction A still — calmo doesn't mean invisible), show only past >600px scroll threshold with fade-in.

### Tab active state should connect to panel

**User observation:**
> Mantener el contraste actual pero añadir un subrayado o borde inferior continuo con la zona de contenido, para que se perciba como "pestaña conectada al panel".

**Direction:** active tab gets a 2px bottom border that connects visually to the panel below — the standard "tab connected" pattern.

## P3 — Optional enhancements

### Scroll spy in sticky

**User observation:**
> Mostrar un indicador de progreso de lectura o el nombre de la sección actual ("Parallelism with multiprocessing") dentro del sticky cuando el usuario está en esa sección.

**Direction:** when scrolled past a heading, surface that heading in the sticky bar. IntersectionObserver per H3.

## Suggested v2.1 milestone scoping

A logical bundle for the first v2.1 phase (working title: "UX polish + sticky header fix"):

- P0 sticky header fix (regression — must land first)
- P1 redundancy + prompt context (rewrite of sticky bar shape)
- P1 sidebar hierarchy (3px border + date groups + density)
- P2 scroll-to-top a11y (one CSS commit + one prop addition)

P2 cost footer relocation is a bigger conversation (Phase 6 D-02 lock revisit) — deserves its own phase or a separate ADR. Suggest deferring to v2.1 phase 2.

P3 scroll spy is enhancement, not fix — defer to v2.1 phase 3 or v2.2.
